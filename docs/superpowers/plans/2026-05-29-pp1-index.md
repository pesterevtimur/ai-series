## Execution mode

Per superpowers:writing-plans skill, после accept'а plan'а будет вопрос:

1. **Subagent-Driven (recommended).** Fresh subagent per task, review между задачами. Лучше изолирует ошибки, но больше overhead на Task tool вызовы.
2. **Inline Execution.** Batch execution с checkpoints. Меньше overhead, но риск load одного контекста большой.

Для ПП1 рекомендуется **inline execution** (вариант 2):
- Phase 1 (lint scripts) — pytest TDD без LLM-критиков → отлично подходит inline.
- Phase 2-3 (.md authoring) — много iteration на формулировки → inline даёт быстрее feedback.
- Phase 4 (smoke-test) — Поток A уже использует Task tool для критиков; дополнительный subagent layer избыточен.

**Тимур решает execution mode на каждой фазе отдельно** — выбор может различаться.
